## 技術突破詳細記錄

### 突破A：M3 Debug Chrome 連線修復
**問題**：Chrome CDP 拒絕外來 WebSocket 連線
```
403 Forbidden: Rejected incoming WebSocket from origin http://localhost:18765
```

**解決**：
```bash
# Chrome 啟動參數必須加：
--remote-allow-origins=*
```

**正確啟動指令**：
```bash
/Applications/Google\ Chrome.app/Contents/MacOS/Google\ Chrome \
  --user-data-dir=/Users/caijunchang/Desktop/chrome-debug-profile \
  --remote-debugging-port=18765 \
  --remote-allow-origins=*
```

**config.yaml 設定**：
```yaml
browser:
  cdp_url: ws://localhost:18765/devtools/browser
```

### 突破B：Veo 3.1 影片下載自動化
**問題**：Gemini Veo 3.1 的 `contribution.usercontent.google.com` 下載需要瀏覽器級 auth，curl 直接下載只會得到 HTML

**解決流程**：
1. CDP `Runtime.evaluate` 深挖 DOM（包括 shadow DOM），找到 `<video>` 標籤的 `src`（格式：`https://contribution.usercontent.google.com/download?c=...&filename=video.mp4&opi=103135050`）
2. CDP `Page.setDownloadBehavior(allow, /Users/caijunchang/Desktop/shots/)`
3. CDP `Page.navigate(video_url)` → Chrome 攔截 `net::ERR_ABORTED` + `isDownload: true` → **自動下載**

**關鍵參數**：
- `browserContextId` 設到瀏覽器層級
- 每次導航前都要先 setDownloadBehavior
- Chrome 會自動帶上所有 page cookies，不需要另外處理 auth

### 突破C：Gemini Prompt 粘貼
**問題**：Gemini 使用 `contenteditable` div，不是標準 `<textarea>`

**解決**：
```javascript
// CDP Runtime.evaluate
element.textContent = promptText;
// 或
document.execCommand('insertText', false, promptText);
```

### 突破D：CDP 跨瀏覽器隔離
**發現**：M3 debug Chrome 和 Hermes CDP browser 是兩個獨立的 Chrome profile
- M3 debug Chrome：`--user-data-dir=/Users/caijunchang/Desktop/chrome-debug-profile`，port 18765
- Hermes CDP：預設沙盒 profile，port 9222
- **不能用同一個 CDP 連線同時操作兩個瀏覽器**

### 突破E：Chrome 下載路徑權限
**問題**：`~/Downloads` 是 root:staff 700，寫不進去

**解決**：用 `Page.setDownloadBehavior(allow, /Users/caijunchang/Desktop/shots/)` 把下載路徑改到桌面 shots 資料夾
